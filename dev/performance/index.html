Pattern

Use R3D.Flat.FlatBuffer (alias for FlatPolytope) and reuse it across iterations:

buf = R3D.Flat.FlatBuffer{3,Float64}(64)
for poly in polys
    R3D.Flat.init_box!(buf, poly.lo, poly.hi)
    R3D.Flat.clip!(buf, poly.planes)
    R3D.Flat.moments!(out, buf, order)
end

This eliminates all per-call allocation. The original "AoS" reference path is still in the package but intentionally unoptimized for cross-validation; it shows up at 5–10× C and should not be used in production.

Where the remaining gap is

reduce order 0 at 1.95× C is the per-call function-dispatch overhead unique to Julia. As order grows the per-call overhead amortizes against more work and the ratio drops to 1.30×. Voxelization inherits this ratio almost exactly because it is dominated by per-leaf moments! calls.

Closing the remaining ~30% would require manual SIMD or function-call elimination at the LLVM level — explicitly out of scope. Where it matters (single small clips in a tight loop), the StaticFlatPolytope variant trims another 20% on init_box! by keeping the polytope on the function frame via MMatrix.
